# zxcvbn-chez

Port of Dropbox's password strength estimator, [zxcvbn](https://github.com/dropbox/zxcvbn), to [Chez Scheme](https://cisco.github.io/ChezScheme/).

> **Status:** This library has not yet been fully reviewed or tested by a human. See [Authorship](#authorship) for details.

## Dependencies

- [Chez Scheme](https://cisco.github.io/ChezScheme/)
- [Akku](https://akkuscm.org/) (package manager)
- [wak-irregex](https://akkuscm.org/packages/wak-irregex/) (irregular expressions)
- [chez-srfi](https://akkuscm.org/packages/chez-srfi/) (SRFI support)

## Installation

Clone the repository and install dependencies with Akku:

```
git clone https://github.com/hinkelman/zxcvbn-chez.git
cd zxcvbn-chez
akku install
```

## Usage

Import the library and call `zxcvbn` with a password string:

```scheme
(import (zxcvbn-chez))

(zxcvbn "password")
```

The `zxcvbn` procedure returns an association list with the following keys:

| Key | Description |
|-----|-------------|
| `"password"` | The password that was evaluated |
| `"guesses"` | Estimated number of guesses needed to crack |
| `"guesses-log10"` | Log base 10 of the guesses estimate |
| `"sequence"` | List of matched patterns found in the password |
| `"crack-time-seconds"` | Alist of estimated crack times (in seconds) under different attack scenarios |
| `"crack-time-display"` | Alist of crack times as human-readable strings |
| `"score"` | Overall strength score from `"0"` (weakest) to `"4"` (strongest) |
| `"feedback"` | Alist with `"warning"` and `"suggestions"` to help the user pick a better password |

### User inputs

You can optionally pass a list of strings (e.g., username, email, site name) that should be treated as easy-to-guess dictionary words:

```scheme
(zxcvbn "travis2026" '("travis" "hinkelman"))
```

### Attack scenarios

The `"crack-time-seconds"` and `"crack-time-display"` keys contain estimates for four attack scenarios:

| Scenario | Guesses per second |
|----------|--------------------|
| `"online-throttling-100-per-hour"` | ~0.028 |
| `"online-no-throttling-10-per-second"` | 10 |
| `"offline-slow-hashing-1e4-per-second"` | 10,000 |
| `"offline-fast-hashing-1e10-per-second"` | 10,000,000,000 |

### Pattern types

The library detects the following patterns in passwords:

- **dictionary** -- common words from frequency lists (English Wikipedia, US TV/film, common passwords, surnames, first names)
- **reverse** -- dictionary words spelled backwards
- **l33t** -- dictionary words with common letter substitutions (e.g., `@` for `a`, `0` for `o`)
- **spatial** -- keyboard walks (e.g., `qwerty`, `zxcvbn`) on QWERTY, Dvorak, and keypad layouts
- **repeat** -- repeated characters or patterns (e.g., `aaa`, `abcabc`)
- **sequence** -- alphabetic or numeric sequences (e.g., `abc`, `654`)
- **regex** -- special patterns like recent years
- **date** -- dates in various formats
